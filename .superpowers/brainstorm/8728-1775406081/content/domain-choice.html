Domain Model: Message as a Reply

How should the Message entity represent its parent?

A: Just the ID

class Message {
  ...
  private readonly parentId?: string;
  ...
}

Simple, but UI needs another query to show what you're replying to.

B: Parent Snippet

class Message {
  ...
  private readonly replyMetadata?: {
    parentId: string;
    parentSnippet: string;
    parentSenderName: string;
  };
}

Self-contained. UI can render the reply context immediately.
