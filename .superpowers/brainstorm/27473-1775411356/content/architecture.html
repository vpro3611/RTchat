Decoupled Reply Architecture

Designing the new independent path for message replies

Endpoint Design Options
Option A: Sub-resource
POST /private/conversation/:id/messages/:msgId/reply

Explicitly links to the parent message in the URL.

Option B: Dedicated Resource
POST /private/conversation/:id/replies

Parent ID is passed in the request body. Clean and flat.

New Components Map

  • Repo: MessageReplyRepositoryPg
  • TxService: ReplyToMessageTxService
  • REST: ReplyToMessageRestController
  • Socket: message:reply (new event)
